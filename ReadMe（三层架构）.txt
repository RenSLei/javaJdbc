

                             关于三层架构(表示层，业务逻辑层，数据访问层)以及本jdbc工程下的包结构说明

1、对于一个系统来讲，Domain对象是非常重要的一个概念，比如对于银行系统来讲，首先需要设计的对象就有用户User等
而对象User的具体细节就是称为User Domain对象;一般来讲一个domain对象就对应一个表。

每个层之间用接口来隔离，而用Domain对象来实现数据的传递，
Domain对象其实与实体概念(需要将数据保存到数据库)差不多。

2、一般而言，Domain对象单独放在一个包中。

3、业务逻辑层和数据访问层之间，用接口隔离，所以必须要写好接口。
一般放在一个dao(DAO :Data Access Objects 数据访问对象是第一个面向对象的接口 ，，数据访问层)包中，是数据访问层的东西。

4、写好了接口后，就要实现接口，一般在接口下面再建一个子包来实现

5、三层架构这样设计的好处是：
业务逻辑层是使用接口，不依赖数据访问层的具体实现，业务逻辑层不关心具体实现是什么，
这样，当我们有不同的实现的时候，不会影响到业务逻辑层，从而实现数据访问层的切换
